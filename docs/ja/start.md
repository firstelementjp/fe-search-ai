# はじめに

FE Search AIは、WordPressにAIによる自然言語検索とチャットUIを追加します。このページでは、最小構成で使い始める流れと主要な概念を説明します。

## セットアップの流れ

1. プラグインをインストールして有効化します。
2. チャットプロバイダーを設定します。
3. Embeddingプロバイダーを設定します。
4. ベクトル保存先としてQdrantを接続します。
5. 同期対象を選び、初回同期を実行します。
6. `[fe-search-ai]` を追加するか、フローティングチャットを有効化します。
7. 実際の質問でテストし、プロンプトやUI設定を調整します。

## 主な概念

### RAG

RAGはRetrieval-Augmented Generationの略です。FE Search AIはまずサイト内の関連コンテンツを検索し、その内容をAIプロバイダーに渡して回答を生成します。

### ハイブリッド検索

ベクトル検索とキーワード検索を組み合わせます。ベクトル検索は意味的に近いコンテンツを見つけやすく、キーワード検索は固有名詞や完全一致に強みがあります。

### 同期

AI回答に利用するには、事前にコンテンツをインデックスする必要があります。同期では、選択したコンテンツをEmbedding化し、Qdrantに保存します。

## 最初におすすめする構成

| 設定 | 推奨値 |
| --- | --- |
| チャットプロバイダー | OpenAI、Google、Anthropicのいずれか |
| Embeddingプロバイダー | OpenAIまたはGoogle |
| ベクトルDB | Qdrant Cloud |
| Rerankプロバイダー | Cohere、任意 |
| 日本語トークナイザー | まずTinySegmenter、必要に応じてYahoo! JAPAN MA API |
