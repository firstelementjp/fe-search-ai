# 設定

FE Search AIの設定は、プロバイダー、同期、表示、高度な設定に分かれています。

## プロバイダー設定

利用するサービスのAPIキーを設定します。

| プロバイダー | 用途 |
| --- | --- |
| OpenAI | チャット補完とEmbedding |
| Google | Geminiチャット補完とEmbedding |
| Anthropic | Claudeチャット補完 |
| Cohere | 任意のRerank |
| Qdrant | ベクトル保存 |
| Yahoo! JAPAN 日本語形態素解析API | 任意の日本語トークナイズ |

## Qdrant設定

Qdrantはサイトコンテンツのベクトルを保存します。

通常、以下を設定します。

- エンドポイントURL
- APIキー
- コレクション名
- 選択したEmbeddingモデルに対応するベクトルサイズ

Qdrant Cloudを使う場合は、同期前にクラスターが有効になっていることを確認してください。

## チャットプロバイダー

回答生成に使うプロバイダーを選びます。無料版ではOpenAI、Google、Anthropicを利用できます。Pro版ではモデル選択やOpenAI互換カスタムエンドポイントを拡張できます。

## Embeddingプロバイダー

コンテンツをベクトル化するプロバイダーを選びます。EmbeddingプロバイダーとQdrantのベクトルサイズは一致している必要があります。

## Rerankプロバイダー

CohereのRerankは任意です。取得されたチャンクを回答生成前に並べ替え、回答品質の改善に役立ちます。

## 同期設定

インデックス対象の投稿タイプを選択します。最初は少ないコンテンツから始め、回答品質を確認してから対象範囲を広げることをおすすめします。

## 表示設定

フロントエンドのチャット文言、色、挨拶文、プレースホルダー、フローティング表示を調整できます。

## プライバシーとレート制限

API利用量を保護するため、レート制限を設定してください。各プロバイダーを有効化する前に、利用規約とプライバシーポリシーを確認してください。
