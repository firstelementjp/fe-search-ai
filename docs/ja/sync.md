# 同期システム

同期システムは、FE Search AIが利用する検索インデックスを構築・維持します。

## 同期で行われること

同期では、選択されたWordPressの投稿、固定ページ、カスタム投稿タイプからコンテンツを抽出します。その後、Embeddingを作成し、検索に必要なメタデータとともにQdrantへ保存します。

## 同期モード

### 手動同期

初回セットアップ時やインデックスを再構築したい場合に実行します。

### 差分同期

変更されたコンテンツを中心に処理し、API利用量と実行時間を抑えます。

### リアルタイム同期

有効化すると、公開・更新されたコンテンツを自動的にインデックスできます。

## おすすめの運用手順

1. まず少ないコンテンツで開始します。
2. 手動同期を実行します。
3. よくある質問で回答をテストします。
4. 投稿タイプや対象範囲を広げます。
5. 品質を確認してからリアルタイム同期を有効化します。

## 日本語コンテンツ

日本語サイトでは、トークナイズ設定がキーワード検索の精度に影響します。TinySegmenterは外部APIなしで利用できます。より高精度な分かち書きが必要な場合は、Yahoo! JAPAN 日本語形態素解析APIを設定してください。

## 同期トラブルの確認項目

- APIキーが有効か確認します。
- Qdrantのエンドポイントとコレクション設定を確認します。
- 共用サーバーではバッチサイズを小さくします。
- PHPメモリ上限と実行時間を確認します。
- 同期が止まる場合はWordPressのデバッグログを確認します。
